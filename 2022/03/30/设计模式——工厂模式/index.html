设计模式——工厂模式

一、简单工厂模式

定义一个工厂类，它可以根据参数的不同返回不同类的实例，被创建的实例通常都具有共同的父类。

ps：传入一个正确的参数，就可以获取所需要的对象，而无需知道其创建细节。

代码实现：

  1. 抽象产品类，整合了各个产品的共同元素：
public abstract class Product{
    //所有产品类的公共业务方法
    public void methodSame(){
        // 公共方法的实现
    }
    // 声明抽象业务方法
    public abstract void methodDiff();
}
  1. 具体产品类，某个具体产品类的实现：
// 继承抽象产品类，从中获得产品类的共同元素
public class ConcreteProduct extends Product{
    // 业务方法的实现
}
  1. 工厂类：
public class Factory{
    // 静态工厂方法
    public static Product getProduct(String arg){
        Product product = null;
        if(arg.equalslgnoreCase("A")){
            product = new ConcreteProductA();
            // 初始化设置product
        }
        else if(arg.aqualslgnoreCase("B")){
            product = new ConcreteProductB();
            // 初始化设置product
        }
        return product
    }
}
  1. 客户端：
public static Client{
    public static void main(String args[]){
        Product product;
        // 通过工厂类创建产品对象
        product = Factory.getProduct("A");
        product.methodSame();
        product.methodDiff();
    }
}

总结：

客户端读入参数，将参数传给工厂，工厂找到参数对应的产品子类，工厂实例化产品父类，再将父类动态绑定到子类实例化的对象上。

二、工厂方法模式

定义一个用于创建对象的接口，但是让子类决定将哪一个类实例化。工厂方法模式让一个类的实例化延迟到其子类。

代码实现：

  1. 抽象工厂类
public interface Factory{
    public Product factoryMethod();
}
  1. 具体工厂类
public class ConcreteFactory implements Factory{
    public Product factoryMethod(){
        return new ConcreteProduct();
    }
}
  1. 客户端
......
Factory factory;
factory = new ConcreteFactory();
Product product;
product = factory.factoryMethod();
......

ps：工厂方法模式和简单工厂模式的区别在于，一个是对产品的共同之处进行了抽象，一个是对产品和工厂的共同之处进行了抽象。
